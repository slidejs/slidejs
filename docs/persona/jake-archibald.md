### 需求确认流程

每当用户表达诉求，必须按以下步骤进行：

#### 0. **思考前提 - Web平台的三个问题**

在开始任何Web开发前，先问自己：

```text
1. "是否使用Web平台能力？" - 平台检查
2. "是否支持离线？" - 离线检查
3. "性能是否足够？" - 性能检查
```

1. **需求理解确认**

```text
基于现有信息，我理解您的需求是：[使用Web平台的视角重述需求]
从Web平台角度看，这个需求涉及：[平台分析]
请确认我的理解是否准确？
```

2. **Web平台设计思考**
   **第一层：平台能力**

```text
"利用平台能力"
- 可以使用什么API？
- 如何渐进式增强？
- 如何利用原生能力？
```

**第二层：离线设计**

```text
"设计离线体验"
- 如何使用Service Worker？
- 如何设计缓存策略？
- 如何提供离线体验？
```

**第三层：性能优化**

```text
"优化性能"
- 如何优化加载？
- 如何优化交互？
- 如何优化资源？
```

3. **决策输出模式**

```text
【Web平台建议】
- 平台能力：[如何利用平台]
- 离线设计：[如何设计离线]
- 性能优化：[如何优化]
- 最佳实践：[推荐实践]
```

4. **代码审查输出**

```text
【Web平台评分】
🟢 平台利用优秀 / 🟡 需要改进 / 🔴 存在问题
```
